Aide & Documentation

Guides fonctionnels · Assistant vocal · NGAP

📖 Fonctionnalités
🔍 Vérifier & corriger — comment ça fonctionne ?
▼
Le bouton 🔍 Vérifier & corriger envoie votre description à l'IA en mode verify avant la cotation. L'IA :
  • Corrige les fautes de frappe et erreurs de reconnaissance vocale
  • Normalise les termes médicaux : "piquer" → "injection SC"
  • Détecte les incohérences NGAP (heure de nuit sans heure précise, enfant sans âge)
  • Suggère les majorations oubliées (IFD si domicile, IK si km présents)
Une fenêtre affiche les corrections proposées pour confirmer leur application.
Cette vérification ne sauvegarde rien en base de données.
💰 Répartition AMO / AMC et DRE — comment ça fonctionne ?
▼
Le workflow calcule automatiquement la répartition après la cotation NGAP :

Calcul AMO :
  • Taux standard : 60 % (ticket modérateur = 40 %)
  • ALD, Maternité, Accident du travail, CSS/C2S : 100 %
Calcul AMC :
  • Part AMC = ticket modérateur, si tiers payant AMO+AMC et mutuelle renseignée
  • Sinon 0 € (patient règle lui-même le ticket)
DRE (Demande de Remboursement Électronique) :
  • Requise si tiers payant (AMO ou AMC), ALD, maternité ou accident du travail
  • Non requise uniquement si paiement direct patient sans exonération
Exemple test réel — Injection insuline 26,35 € · Patient ALD · Tiers payant AMO+AMC :
part_amo = 26,35 € (100 % ALD) · part_amc = 0 € · part_patient = 0 € · dre_requise = true
Ces valeurs sont stockées en BDD et apparaissent sur la feuille de soins PDF.
🎤 Assistant vocal — toutes les commandes disponibles
▼
L'Assistant vocal (bouton 🎤 Vocal en haut à droite) écoute en continu. Seuil de confiance : 0,55.

Navigation & actions :
  • "nouveau patient" / "patient suivant" / "réinitialiser" — vide tous les champs
  • "vider les soins" / "effacer le texte" — vide uniquement la description
  • "générer la facture" / "calculer" / "facturer" / "valider" — lance la cotation
  • "vérifier" / "corriger" — ouvre la fenêtre de vérification
  • "stop assistant" / "arrêter" / "couper" — désactive l'assistant
Saisie des champs :
  • "nom du patient [nom]" — remplit le nom patient
  • "prénom [prénom]" / "s'appelle [prénom]" — remplit le prénom
  • "sécurité sociale [chiffres]" — remplit le NIR
  • "ADELI [chiffres]" — remplit le N° ADELI
  • "date du soin" / "aujourd'hui" — met la date du jour
  • "heure du soin à 22h" — remplit le champ heure
  • "tiers payant" → AMO ; "tiers payant complet" → AMO+AMC
  • "ALD" / "exonéré" / "cent pour cent" → exonération ALD 100 %
  • Tout texte avec un mot-clé NGAP → ajouté automatiquement au champ soins
Normalisation médicale — termes reconnus :
  • "piquer" / "administrer insuline" → "injection insuline SC"
  • "prise de sang" / "bilan sanguin" → "prélèvement sanguin"
  • "toilette totale" / "bain complet" → "toilette complète"
  • "grabataire" / "alité" / "immobilisé" → "patient grabataire"
  • "chez le patient" / "au domicile" → "domicile"
  • "vingt-deux heures" → "22h" · "midi" → "12h"
  • "dix kilomètres" → "10 km"
🏥 Normalisation médicale — termes NGAP reconnus
▼
La fonction normalizeMedical() transforme la voix brute en termes NGAP propres :
  • "piquer" / "administrer insuline" → "injection insuline SC"
  • "prise de sang" / "bilan sanguin" → "prélèvement sanguin"
  • "toilette totale" / "bain complet" → "toilette complète"
  • "grabataire" / "alité" / "immobilisé" → "patient grabataire"
  • "très dépendant" / "dépendance sévère" → "dépendance lourde"
  • "chez le patient" / "au domicile" → "domicile"
  • "vingt-deux heures" → "22h" · "midi" → "12h" · "minuit" → "0h"
  • "dix kilomètres" → "10 km"
  • "coordination" / "équipe soignante" → "coordination soignante"
  • Chiffres écrits de 0 à 90 → convertis en nombres
