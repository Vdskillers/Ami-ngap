🏗️ Architecture générale
Quels sont les composants de l'infrastructure ?
▼
InfirmierFacture / AMI repose sur 4 composants :
  • 🌐 Frontend — GitHub Pages · Interface statique
  • ⚙️ API Backend — Cloudflare Worker · Auth, profil, proxy n8n · < 50ms
  • 🗄️ Base de données — Supabase (PostgreSQL) · Comptes, sessions, profils
  • 🤖 Agent IA — n8n sur Render · Workflow Grok-3, cotation NGAP, RAG, historique
🏠 Vue d'ensemble
Comment fonctionne l'IA de cotation ?
▼
L'IA analyse votre description et détecte automatiquement :
  • 🩺 Les actes (injection SC/IM/IV, pansement, toilette, perfusion, prélèvement…)
  • 🕒 L'heure — majorations nuit (9,15 €) ou nuit profonde 23h-5h (18,30 €)
  • 📍 Le lieu (domicile → IFD automatique)
  • 🚗 La distance (km → IK automatique)
  • 👶 L'âge (enfant < 7 ans → MIE obligatoire)
  • 🧠 La dépendance (BSA / BSB / BSC selon niveau)
  • 📅 Le jour (dimanche/férié → majoration 8,50 €)
✨ RAG NGAP — avant chaque cotation, toutes les règles NGAP pertinentes sont récupérées via BM25 adaptatif et injectées dans le prompt.
Validation des champs — faut-il tout remplir ?
▼
Validation à 3 niveaux non-bloquante :
  • Bloquant strict : la description des soins est le seul champ vraiment obligatoire
  • Confirmation : si nom infirmière, nom patient ou NIR manquent, une fenêtre demande confirmation
  • Léger : si champs secondaires manquent (ADELI, RPPS, DDN…), confirmation légère

Règles de coefficients et cumuls
▼
  • Acte principal : coefficient 1 (tarif plein)
  • Actes secondaires : coefficient 0,5 (demi-tarif)
  • Majorations (nuit, dimanche, IFD, IK, MIE, MCI) : toujours coefficient 1
  • AIS + BSx : INTERDIT — alerte URSSAF déclenchée
  • BSA + BSB + BSC : s'excluent mutuellement
  • Un seul acte principal par passage
Optimisations légales proposées par l'IA
▼
  • AIS → BSA (+10,35 €) si dépendance légère détectée
  • BSA → BSB (+5,20 €) si dépendance intermédiaire
  • BSB → BSC (+10,50 €) si dépendance lourde
  • + MCI (+5,00 €) si coordination pluridisciplinaire
  • + MIE (+3,15 €) si enfant < 7 ans non coté
  • + IFD (+2,75 €) si domicile mentionné sans IFD
  • + IK si distance mentionnée sans IK
🚨 Alertes URSSAF
Les 9 alertes de conformité détectées automatiquement
▼
  1. AIS + BSx cotés ensemble (cumul interdit)
  2. Deux forfaits BSx cotés ensemble
  3. Majoration nuit sans heure explicite
  4. Majoration dimanche sans jour explicite
  5. IFD cotée sans visite à domicile
  6. IK cotée sans distance mentionnée
  7. MIE appliquée à un patient ≥ 7 ans
  8. Acte secondaire au coefficient 1 au lieu de 0,5
  9. AMI coté pour un acte relevant de l'AIS
📊 Feuille de soins PDF
Que contient la feuille de soins PDF ?
▼
  • En-tête : NGAP 2026, N° facture, date et heure du soin
  • Professionnel de santé : nom, N° ADELI, N° RPPS, structure
  • Patient : nom, date de naissance, NIR, exonération
  • Couverture maladie : caisse AMO, mutuelle AMC, mode règlement
  • Tableau des actes : libellé, code NGAP, coefficient, tarif, montant
  • Répartition : part AMO, part AMC, reste patient
  • Zone signature avec ADELI et RPPS
Répartition AMO / AMC et DRE — comment ça fonctionne ?
▼
AMO : 60 % standard · 100 % si ALD, Maternité, AT, CSS/C2S
AMC : = ticket modérateur si tiers payant AMO+AMC avec mutuelle, sinon 0 €
DRE requise si tiers payant, ALD, maternité ou accident du travail

Exemple — Injection 26,35 € · ALD · Tiers AMO+AMC :
part_amo = 26,35 € · part_amc = 0 € · part_patient = 0 € · dre_requise = true
🎤 Dictée vocale & Assistant
Dictée soins — fonctionnement
▼
  • 3 alternatives : sélectionne la transcription la plus fiable
  • Seuil de confiance 0,65 : segments peu clairs ignorés
  • Normalisation NGAP automatique via normalizeMedical()
  • Redémarrage automatique après pause
  • Compatible : Chrome, Edge uniquement
Assistant vocal — toutes les commandes
▼
Navigation :
  • "nouveau patient" / "réinitialiser" — vide tous les champs
  • "générer la facture" / "calculer" / "valider" — lance la cotation
  • "vérifier" / "corriger" — ouvre la fenêtre de vérification
  • "patient terminé" (pilotage live)
  • "stop assistant" / "arrêter" — désactive
Saisie champs :
  • "nom du patient [nom]" · "prénom [prénom]" · "sécurité sociale [chiffres]"
  • "date du soin" / "aujourd'hui" · "heure du soin à 22h"
  • "tiers payant" → AMO · "tiers payant complet" → AMO+AMC
  • "ALD" / "exonéré" → exonération ALD 100 %
Normalisation médicale :
  • "piquer" → "injection SC" · "prise de sang" → "prélèvement sanguin"
  • "toilette totale" → "toilette complète" · "grabataire" → "patient grabataire"
  • "vingt-deux heures" → "22h" · "midi" → "12h"
  • "dix kilomètres" → "10 km" · "coordination" → "coordination soignante"
  • Chiffres écrits 0–90 → convertis en nombres
🚗 Tournées & Planning
Simulation tournée — comment l'utiliser ?
▼
Décrivez plusieurs patients — l'IA calcule le revenu estimé de la tournée avec NGAP complet. Ce mode ne sauvegarde rien (_skip_db: true). Résultat : revenu total, nombre de patients, revenu moyen, revenu horaire estimé, optimisations légales.
Optimisation de tournée — comment ça fonctionne ?
▼
Analyse votre liste de patients et propose :
  • L'ordre optimal de passage (patients proches regroupés)
  • Le revenu estimé de la tournée optimisée
  • Des conseils (soins lourds en début, regrouper les nuits…)
Planning semaine — que génère-t-il ?
▼
Planning hebdomadaire optimisé : 8–15 patients/jour · Soins lourds le matin · Regroupement géographique · Équilibre des revenus. Résultat : planning jour par jour avec revenus et temps estimés.
🧠 Intelligence RAG NGAP
Qu'est-ce que le RAG NGAP ?
▼
Le RAG (Retrieval-Augmented Generation) enrichit l'IA avec une base de connaissances réglementaires avant chaque cotation :
  1. Description analysée par BM25
  2. Tous les chunks pertinents récupérés (seuil adaptatif ≥ 30 % du meilleur score)
  3. 3 chunks toujours inclus : ngap_001 (coefficients) · ngap_002 (cumuls interdits) · ngap_016 (alertes URSSAF)
  4. Si aucun match : 20 chunks injectés en intégralité
20 chunks couvrant le Titre XVI NGAP métropole 2026 : AMI, AIS, BSA/B/C, IFD, IK, MCI, MIE, majorations horaires, optimisations légales.
⚙️ Workflow technique
Comment fonctionne le workflow n8n de cotation ?
▼
Le workflow AI Agent AMI s'exécute en 9 étapes :
  1. Webhook /ami-calcul — reçoit les données du formulaire
  2. Init BDD — ping + migrations colonnes
  3. RAG NGAP Retriever — BM25 seuil adaptatif
  4. AI Agent (Grok-3) — prompt enrichi RAG, retourne JSON coté
  5. Parser résultat IA — calcule AMO/AMC/DRE
  6. Mode spécial ? — bypass BDD pour verify/tournée/planning
  7. Sauvegarder en BDD — INSERT complet
  8. Fusionner réponse — JSON final
  9. Respond to Webhook — retourne la réponse
Temps typique : 15–35 secondes (temps de réponse Grok-3).
🔧 Problèmes courants
La facture ne se génère pas — que faire ?
▼
  1. Vérifiez que le champ description des soins est rempli (seul champ bloquant)
  2. Vérifiez votre connexion internet
  3. Le serveur n8n sur Render peut être en veille — attendez 30 secondes et réessayez
  4. Utilisez 🔍 Vérifier & corriger pour valider la description avant génération
La dictée vocale ne fonctionne pas
▼
Prérequis : Google Chrome ou Microsoft Edge · Autorisation microphone · Connexion internet active.
Non supporté Firefox, Safari, navigateurs mobiles non-Chromium.
La cotation semble incorrecte
▼
  1. Description trop vague : utilisez 🔍 Vérifier & corriger avant de générer
  2. Heure ou date manquante : renseignez les champs pour les majorations
  3. Cas hors périmètre : le RAG couvre le Titre XVI métropole. Pour les DOM/TOM, les règles peuvent différer.
En cas de doute, vérifiez sur ameli.fr.
