🏗️ Architecture générale
Quels sont les composants de l'infrastructure ?
▼
AMI repose sur 4 composants :
  • 🌐 Frontend PWA — GitHub Pages · Interface statique · installable sur mobile et desktop
  • ⚙️ API Backend — Cloudflare Worker v6.1 · Auth, profil, isolation données, proxy n8n · < 50ms
  • 🗄️ Base de données — Supabase (PostgreSQL EU) · Comptes, sessions, cotations chiffrées, logs
  • 🤖 Agent IA — n8n sur Render · Workflow Grok-3, cotation NGAP, RAG BM25, historique
Les données de santé (patients, signatures) sont stockées exclusivement sur votre appareil en IndexedDB chiffré AES-256 — elles ne transitent jamais par nos serveurs.
Quels modules composent l'application ?
▼
  • 🏠 Dashboard — statistiques, anomalies IA, prévisions revenus
  • ⚡ Cotation NGAP — saisie, vérification IA, génération facture, impression
  • 👥 Carnet patients — CRUD local chiffré, notes soins, ordonnances, export
  • 📋 Import calendrier — ICS, CSV, JSON, texte libre, géocodage auto
  • 🗺️ Tournée IA — optimisation OSRM, carte Leaflet, GPS, estimation CA
  • 🚦 Pilotage live — suivi en temps réel, auto-facturation, détection retard
  • 🚗 Mode Uber médical — patient suivant dynamique, tracking GPS continu
  • ✍️ Signatures électroniques — canvas tactile/souris, stockage chiffré local
  • 🤖 Copilote IA — assistant conversationnel contextuel
  • 📊 Rapport mensuel — génération HTML/PDF, santé système
  • 💶 Trésorerie — suivi paiements, export comptable, checklist CPAM
  • 🎤 Assistant vocal — NLP avancé, commandes mains-libres, dictée médicale
  • 📬 Contact admin — messagerie infirmière → administrateur
  • 🔐 Sécurité RGPD — consentement, PIN, export/purge données
🔒 Sécurité & Données personnelles
Où sont stockées mes données patients ?
▼
🛡️ Privacy by Design — Les données de santé sont stockées exclusivement sur votre terminal et ne sont jamais transmises à nos serveurs.
  • Patients & notes soins → IndexedDB local · chiffrement AES-256 · clé dérivée de votre session
  • Signatures électroniques → IndexedDB local · chiffrement AES-256 · jamais synchronisées
  • Cotations → Supabase EU · chiffrement de champ AES-256-GCM côté serveur
  • Logs d'accès → IndexedDB local · métadonnées uniquement · purge auto 90 jours
Isolation des données entre utilisateurs
▼
  • Chaque infirmière ne voit que ses propres données — isolation stricte par infirmiere_id côté serveur
  • Les admins peuvent tester toutes les fonctionnalités avec leurs propres patients de test — ils n'ont jamais accès aux données des infirmières
  • Le panneau admin affiche uniquement les statistiques globales et le nom/prénom des infirmières — aucun email, patient, NIR ni donnée médicale visible
  • Les comptes admins sont invisibles dans le panneau d'administration
Chiffrement — comment ça fonctionne ?
▼
  • Côté client : Web Crypto API · AES-256-GCM · clé dérivée du token JWT de session
  • Côté serveur : crypto.createCipheriv('aes-256-gcm', key, iv) sur Cloudflare Worker
  • Mots de passe : hashés bcrypt via Supabase Auth
  • Sessions : JWT sécurisés, vérifiés à chaque requête
  • PIN local : verrouillage automatique après 10 minutes d'inactivité
Mes droits RGPD
▼
  • 📥 Export — toutes vos données en JSON via Sécurité → "Exporter mes données"
  • 🗑️ Effacement — données locales via "Effacer données locales" · données serveur via "Supprimer mon compte"
  • ✏️ Modification — profil modifiable dans Mon Compte
  • ⚠️ Révocation consentement — disponible dans Sécurité → "Révoquer mon consentement"
⚡ Cotation NGAP
Comment fonctionne l'IA de cotation ?
▼
L'IA analyse votre description et détecte automatiquement :
  • 🩺 Les actes (injection SC/IM/IV, pansement, toilette, perfusion, prélèvement…)
  • 🕒 L'heure — majorations nuit (9,15 €) ou nuit profonde 23h-5h (18,30 €)
  • 📍 Le lieu (domicile → IFD automatique)
  • 🚗 La distance (km → IK automatique)
  • 👶 L'âge (enfant < 7 ans → MIE obligatoire)
  • 🧠 La dépendance (BSA / BSB / BSC selon niveau)
  • 📅 Le jour (dimanche/férié → majoration 8,50 €)
✨ RAG NGAP — avant chaque cotation, toutes les règles NGAP pertinentes sont récupérées via BM25 adaptatif et injectées dans le prompt.
Validation des champs — faut-il tout remplir ?
▼
Validation à 3 niveaux non-bloquante :
  • Bloquant strict : la description des soins est le seul champ vraiment obligatoire
  • Confirmation : si nom infirmière, nom patient ou NIR manquent, une fenêtre demande confirmation
  • Léger : si champs secondaires manquent (ADELI, RPPS, DDN…), confirmation légère
Cotation hors-ligne — comment ça fonctionne ?
▼
Quand vous n'avez pas internet, les cotations sont mises en file d'attente localement.
Dès que la connexion est rétablie, elles sont synchronisées automatiquement avec le serveur.
Un badge indique le nombre de cotations en attente.

Règles de coefficients et cumuls
▼
  • Acte principal : coefficient 1 (tarif plein)
  • Actes secondaires : coefficient 0,5 (demi-tarif)
  • Majorations (nuit, dimanche, IFD, IK, MIE, MCI) : toujours coefficient 1
  • AIS + BSx : INTERDIT — alerte URSSAF déclenchée
  • BSA + BSB + BSC : s'excluent mutuellement
  • Un seul acte principal par passage
Optimisations légales proposées par l'IA
▼
  • AIS → BSA (+10,35 €) si dépendance légère détectée
  • BSA → BSB (+5,20 €) si dépendance intermédiaire
  • BSB → BSC (+10,50 €) si dépendance lourde
  • + MCI (+5,00 €) si coordination pluridisciplinaire
  • + MIE (+3,15 €) si enfant < 7 ans non coté
  • + IFD (+2,75 €) si domicile mentionné sans IFD
  • + IK si distance mentionnée sans IK
🚨 Alertes URSSAF
Les 9 alertes de conformité détectées automatiquement
▼
  1. AIS + BSx cotés ensemble (cumul interdit)
  2. Deux forfaits BSx cotés ensemble
  3. Majoration nuit sans heure explicite
  4. Majoration dimanche sans jour explicite
  5. IFD cotée sans visite à domicile
  6. IK cotée sans distance mentionnée
  7. MIE appliquée à un patient ≥ 7 ans
  8. Acte secondaire au coefficient 1 au lieu de 0,5
  9. AMI coté pour un acte relevant de l'AIS
👥 Carnet patients
Comment fonctionne le carnet patients ?
▼
Le carnet patients est 100 % local et chiffré. Aucune donnée patient ne quitte votre appareil.
  • Fiche patient : nom, prénom, date naissance, téléphone, NIR, adresse structurée, ordonnance
  • Notes de soins : 4 catégories (général · accès · médical · urgent) · CRUD complet
  • Cotations liées : historique des cotations par patient
  • Géocodage automatique : coordonnées GPS calculées à la saisie de l'adresse
  • Navigation GPS : bouton de navigation directe vers le domicile du patient
  • Export : toutes vos fiches en JSON depuis la fiche patient
  • Sync PC ↔ mobile : sauvegarde/restauration chiffrée via les routes patients-push/pull
Correction de position GPS — comment corriger une adresse ?
▼
Si le géocodage automatique place le marqueur au mauvais endroit :
  1. Ouvrez la fiche patient → cliquez "Corriger position"
  2. Tapez ou glissez le marqueur sur la carte
  3. Cliquez "Valider la position"
La correction est mémorisée et appliquée automatiquement aux prochaines saisies similaires (saveLearnedCorrection).

📄 Feuille de soins PDF
Que contient la feuille de soins PDF ?
▼
  • En-tête : NGAP 2026, N° facture séquentiel, date et heure du soin
  • Professionnel de santé : nom, N° ADELI, N° RPPS, structure
  • Patient : nom, date de naissance, NIR, exonération
  • Couverture maladie : caisse AMO, mutuelle AMC, mode règlement
  • Tableau des actes : libellé, code NGAP, coefficient, tarif, montant
  • Répartition : part AMO, part AMC, reste patient
  • Zone signature avec ADELI et RPPS
Répartition AMO / AMC et DRE — comment ça fonctionne ?
▼
AMO : 60 % standard · 100 % si ALD, Maternité, AT, CSS/C2S
AMC : = ticket modérateur si tiers payant AMO+AMC avec mutuelle, sinon 0 €
DRE requise si tiers payant, ALD, maternité ou accident du travail

Exemple — Injection 26,35 € · ALD · Tiers AMO+AMC :
part_amo = 26,35 € · part_amc = 0 € · part_patient = 0 € · dre_requise = true

🧠 Intelligence RAG NGAP
Qu'est-ce que le RAG NGAP ?
▼
Le RAG (Retrieval-Augmented Generation) enrichit l'IA avec une base de connaissances réglementaires avant chaque cotation :
  1. Description analysée par BM25
  2. Tous les chunks pertinents récupérés (seuil adaptatif ≥ 30 % du meilleur score)
  3. 3 chunks toujours inclus : ngap_001 (coefficients) · ngap_002 (cumuls interdits) · ngap_016 (alertes URSSAF)
  4. Si aucun match : 20 chunks injectés en intégralité
20 chunks couvrant le Titre XVI NGAP métropole 2026 : AMI, AIS, BSA/B/C, IFD, IK, MCI, MIE, majorations horaires, optimisations légales.
⚙️ Workflow technique
Comment fonctionne le workflow n8n de cotation ?
▼
Le workflow AI Agent AMI s'exécute en 9 étapes :
  1. Webhook /ami-calcul — reçoit les données du formulaire
  2. Init BDD — ping + migrations colonnes
  3. RAG NGAP Retriever — BM25 seuil adaptatif
  4. AI Agent (Grok-3) — prompt enrichi RAG, retourne JSON coté
  5. Parser résultat IA — calcule AMO/AMC/DRE
  6. Mode spécial ? — bypass BDD pour verify/tournée/planning
  7. Sauvegarder en BDD — INSERT complet
  8. Fusionner réponse — JSON final
  9. Respond to Webhook — retourne la réponse
Temps typique : 15–35 secondes (temps de réponse Grok-3). Si n8n ne répond pas, un fallback local calcule la cotation automatiquement.

🔧 Problèmes courants
La facture ne se génère pas — que faire ?
▼
  1. Vérifiez que le champ description des soins est rempli (seul champ bloquant)
  2. Vérifiez votre connexion internet
  3. Le serveur n8n sur Render peut être en veille — attendez 30 secondes et réessayez
  4. Utilisez 🔍 Vérifier & corriger pour valider la description avant génération
  5. En cas d'échec persistant, la cotation est sauvegardée en file d'attente et synchronisée dès que la connexion est rétablie
La dictée vocale ne fonctionne pas
▼
Prérequis : Google Chrome ou Microsoft Edge · Autorisation microphone · Connexion internet active.
Non supporté Firefox, Safari, navigateurs mobiles non-Chromium.
La cotation semble incorrecte
▼
  1. Description trop vague : utilisez 🔍 Vérifier & corriger avant de générer
  2. Heure ou date manquante : renseignez les champs pour les majorations
  3. Cas hors périmètre : le RAG couvre le Titre XVI métropole. Pour les DOM/TOM, les règles peuvent différer.
En cas de doute, vérifiez sur ameli.fr.
Mes patients ne s'affichent plus sur un autre appareil
▼
Le carnet patients est stocké localement sur chaque appareil. Pour synchroniser :
  1. Sur l'appareil source → onglet Sécurité → "Sauvegarder patients"
  2. Sur le nouvel appareil → "Restaurer patients"
La synchronisation est chiffrée de bout en bout.
L'application me demande un PIN
▼
Le verrouillage automatique PIN s'active après 10 minutes d'inactivité. C'est une mesure de sécurité RGPD pour protéger vos données patients.
Configurez votre PIN dans Sécurité → Configurer un PIN.
